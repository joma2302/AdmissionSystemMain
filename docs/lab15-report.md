# ЗВІТ З НАВЧАЛЬНОЇ ПРАКТИКИ
## Лабораторна робота №15
### Тема: Розробка та тестування представлень сторінок адміністративної частини інформаційної системи

---

### 1. Вступ
**Напрям дослідження:** Проектування користувацьких інтерфейсів (UI/UX) для складних інформаційних систем, розробка динамічних веб-шаблонів та інтеграція фронтенд-частини з серверною логікою на базі Spring Boot та FreeMarker.

**Мета роботи:** Створити зручний, інформативний та адаптивний інтерфейс для адміністративного модуля системи "Приймальна комісія", забезпечити візуалізацію аналітичних даних та реалізувати механізми керування процесами вступу.

**Завдання:**
1. Розробити макет головної панелі адміністратора (Dashboard) зі зведеною статистикою.
2. Створити інтерфейси для керування списками абітурієнтів та їхніми заявками.
3. Реалізувати сторінку детальної інформації про абітурієнта з відображенням оцінок та статусів заяв.
4. Модернізувати форми керування факультетами та вимогами до вступників.
5. Провести тестування розроблених шаблонів на коректність відображення даних та працездатність Java-контролерів.

---

### 2. Лаконічний опис результатів виконання
У ході виконання роботи було спроектовано та впроваджено комплексний веб-інтерфейс адміністративної частини.

**Ключові результати:**
- **Dashboard (Панель приладів):** Впроваджено систему карток метрик, що відображають загальну кількість абітурієнтів, заявок, зарахованих та вільних місць. Додано таблицю попиту з індикаторами заповненості (progress bars).
- **Керування абітурієнтами:** Створено табличний інтерфейс із візуальною ідентифікацією (аватари) та можливістю переходу до повного досьє абітурієнта.
- **Деталізація профілю:** Реалізовано сторінку `applicant-details.ftl`, яка консолідує дані про особу, її бали ЗНО/НМТ та історію поданих заявок.
- **Фільтрація та модерація:** Впроваджено форму фільтрації заявок за факультетами та статусами, а також механізм ручного оновлення статусу заявки безпосередньо з таблиці.
- **Адаптивний дизайн:** Використано сучасні CSS-стилі для забезпечення коректного відображення на різних типах пристроїв.

### 3. Текст програми з коментарями

#### 3.1. Логіка підготовки даних для адмін-панелі (AdminController.java)
```java
/**
 * Контролер забезпечує передачу аналітичних даних до шаблонів FreeMarker.
 */
@Controller
public class AdminController {
    // ... залежності ...

    /** Відображення головної панелі зі статистикою */
    @GetMapping("/admin")
    public String dashboard(Model model) {
        // Отримання загальних статистичних показників (кількість заявок, місць тощо)
        model.addAttribute("stats", admissionService.getDashboardStats());
        // Отримання даних про конкурс на кожен факультет
        model.addAttribute("facultyDemand", admissionService.getFacultyDemand());
        return "admin/dashboard";
    }

    /** Детальна інформація про абітурієнта за його ідентифікатором */
    @GetMapping("/admin/applicants/{id}")
    public String showApplicantDetails(@PathVariable String id, Model model) {
        applicantService.getApplicant(id).ifPresent(applicant -> {
            model.addAttribute("applicant", applicant);
            // Завантаження оцінок та поданих заяв для конкретної особи
            model.addAttribute("grades", gradeRepository.findByApplicantId(id));
            model.addAttribute("applications", applicationRepository.findByApplicantId(id));
        });
        return "admin/applicant-details";
    }
}
```

#### 3.2. Фрагмент шаблону Dashboard (dashboard.ftl)
```html
<#-- Використання макросу сторінки для збереження єдиного стилю -->
<@layout.page title="Адмін-панель">
    <div class="metric-grid">
        <#-- Картка з кількістю абітурієнтів -->
        <div class="metric-card">
            <div class="metric-icon">👥</div>
            <div class="metric-content">
                <span>Абітурієнтів</span>
                <strong>${stats.applicants()}</strong>
            </div>
        </div>
        <#-- Картка зі статусом зарахування -->
        <div class="metric-card success">
            <div class="metric-icon">✅</div>
            <div class="metric-content">
                <span>Зараховано</span>
                <strong>${stats.admitted()}</strong>
            </div>
        </div>
    </div>

    <#-- Таблиця попиту за спеціальностями -->
    <div class="content-card">
        <h2>Попит за спеціальностями</h2>
        <table class="data-table">
            <#list facultyDemand as item>
                <tr>
                    <td><strong>${item.facultyName()}</strong></td>
                    <td>
                        <#-- Візуальний прогрес заповнення місць -->
                        <div class="progress-container">
                            <div class="progress-bar" style="width: ${(item.admitted() / item.seats() * 100)}%"></div>
                            <span class="progress-text">${item.admitted()} / ${item.seats()}</span>
                        </div>
                    </td>
                </tr>
            </#list>
        </table>
    </div>
</@layout.page>
```

### 4. Вхідні та вихідні дані програми

**Вхідні дані:**
- Об'єкти `Applicant`, `Application`, `Faculty`, отримані з бази даних MySQL через відповідні репозиторії.
- Статистичні записи (records) `AdminDashboardStats` та `FacultyDemand`.
- Параметри HTTP-запитів для фільтрації списків (наприклад, `facultyName`, `status`).

**Вихідні дані:**
- Згенеровані HTML-сторінки з динамічним контентом.
- Візуальні графічні елементи (прогрес-бари, кольорові бейджі статусів).
- HTTP-відповіді з результатами операцій (успішне оновлення статусу, повідомлення про помилки).
- Логи сервера про доступ до адміністративних ресурсів.

**Аналіз результатів:**
Розроблені представлення забезпечують високий рівень наочності процесів вступної кампанії. Впровадження сторінки деталей абітурієнта дозволило адміністраторам приймати більш зважені рішення при ручному коригуванні статусів. Тестування показало, що сторінки коректно обробляють відсутність даних (порожні списки) та забезпечують швидку навігацію між пов'язаними сутностями.

Макети сторінок відповідають сучасним стандартам UX/UI, забезпечуючи зручне керування великими обсягами даних. Процес тестування підтвердив цілісність системи та правильність взаємодії між фронтендом та бекендом. Створений інструментарій є критично важливим для ефективної роботи членів приймальної комісії.
