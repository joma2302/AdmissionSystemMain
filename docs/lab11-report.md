# ЗВІТ З НАВЧАЛЬНОЇ ПРАКТИКИ
## Лабораторна робота №11
### Тема: Налаштування адміністративної частини веб-застосунку та розмежування прав доступу


---

### 1. Вступ
**Напрям дослідження:** Веб-розробка на базі Spring Framework, забезпечення безпеки інформаційних систем.

**Мета роботи:** Реалізувати адміністративну частину інформаційної системи "Приймальна комісія", забезпечити розмежування ролей користувачів та захист конфіденційних даних.

**Завдання:**
1. Впровадити ролі `ADMIN` та `USER` (APPLICANT).
2. Налаштувати Spring Security для обмеження доступу до адміністративних сторінок.
3. Створити окремий інтерфейс (Dashboard) для адміністратора.
4. Реалізувати механізм перевірки прав доступу (Access Denied) та коректних редиректів.
5. Забезпечити відображення UI елементів відповідно до прав користувача.

---

### 2. Опис результатів виконання
В ході виконання роботи було модернізовано систему безпеки та інтерфейс користувача.

**Ключові досягнення:**
- **Безпека:** Налаштовано `SecurityFilterChain`, який блокує доступ до шляхів `/admin/**` та `/admission/**` для користувачів без ролі `ADMIN`.
- **Контроль доступу:** Додано обробку помилки 403 (Access Denied), що перенаправляє користувача на сторінку логіну з відповідним повідомленням.
- **Адміністративна панель:** Створено `AdminController` та шаблон `admin/dashboard.ftl`, що дозволяє керувати абітурієнтами, факультетами та проводити зарахування.
- **Динамічний UI:** Оновлено базовий шаблон `layout.ftl`, де навігація змінюється залежно від ролі авторизованого користувача.

### 3. Текст програми з коментарями

#### Конфігурація безпеки (SecurityConfig.java)
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http,
                                                   DaoAuthenticationProvider daoAuthenticationProvider) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                // Публічні сторінки доступні всім
                .requestMatchers("/", "/signup", "/login", "/error", "/css/**", "/js/**").permitAll()
                // Захист адміністративних ендпоінтів - ТІЛЬКИ ДЛЯ ADMIN
                .requestMatchers("/admission/**").hasRole("ADMIN")
                .requestMatchers("/admin/**").hasRole("ADMIN")
                // Решта сторінок потребує звичайної авторизації
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login") // Власна сторінка входу
                .defaultSuccessUrl("/", true)
                .failureUrl("/login?error=true")
                .permitAll()
            )
            .exceptionHandling(ex -> ex
                .accessDeniedPage("/login?error=denied") // Редирект при спробі несанкціонованого доступу
            )
            .logout(logout -> logout
                .logoutUrl("/logout")
                .logoutSuccessUrl("/login?logout=true")
                .permitAll()
            )
            .authenticationProvider(daoAuthenticationProvider);

        return http.build();
    }
}
```

#### Навігаційне меню (layout.ftl)
```html
<div class="nav-links">
    <a href="/">Головна</a>

    <#-- Перевірка авторизації через контекст Spring Security -->
    <#if isAuthenticated>
        <#-- Визначення ролі користувача -->
        <#assign isAdmin = false>
        <#if auth.authorities??>
            <#list auth.authorities as authItem>
                <#if authItem.authority == "ROLE_ADMIN">
                    <#assign isAdmin = true>
                </#if>
            </#list>
        </#if>

        <#-- Відображення меню відповідно до ролі -->
        <#if isAdmin>
            <a href="/admin" style="color: #60a5fa; font-weight: bold;">Адмін-панель</a>
        <#else>
            <a href="/register">Реєстрація</a>
            <a href="/apply">Подати заявку</a>
            <a href="/applications">Мої заявки</a>
        </#if>
        <a href="/ranking">Рейтинг</a>
    </#if>
</div>
```

### 4. Вхідні та вихідні дані програми

**Вхідні дані:**
- Облікові дані користувача (username, password).
- База даних з ролями (ADMIN, APPLICANT).
- Запити на HTTP-ресурси (URL).

**Вихідні дані:**
- Для ролі **ADMIN**: Доступ до сторінки `/admin`, можливість бачити кнопку "Адмін-панель", доступ до статистики системи.
- Для ролі **USER (APPLICANT)**: Доступ до подачі заявок, відсутність адмін-кнопок, при спробі зайти на `/admin` — редирект на логін з помилкою доступу.
- Логи системи (`admission-system.log`), що фіксують успішні входи та спроби порушення прав.

---

### 5. Змістовний аналіз та висновки

**Аналіз результатів:**
Проектування системи розмежування ролей дозволило ізолювати критичні функції управління системою від звичайних користувачів. Використання Spring Security забезпечило надійний захист на рівні URL-фільтрів. Тестування показало, що:
1. Користувач `admin` має повний контроль над факультетами та заявками.
2. Звичайний абітурієнт не бачить адміністративних елементів і не може отримати доступ до них за прямим посиланням.
3. Система коректно обробляє сесії та вихід з акаунту.

**Висновки:**
В ході виконання лабораторної роботи було успішно реалізовано адміністративну частину веб-застосунку. Навченося працювати з механізмами авторизації та автентифікації Spring Security, використовувати шаблонізатор FreeMarker для динамічного відображення контенту та забезпечувати безпеку бізнес-логіки. Система готова до експлуатації в багатокористувацькому середовищі з чітким розділенням відповідальності.
