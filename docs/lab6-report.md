# Звіт з лабораторної роботи № 6

**Тема:** Створення веб-сторінок та контролерів  
**Мета:** Набуття навичок розробки користувацького інтерфейсу за допомогою шаблонізаторів та створення MVC-контролерів для обробки бізнес-логіки в середовищі Spring Boot.

---

## 1. Вступ

На даному етапі розробки інформаційної системи «Приймальна комісія» було реалізовано рівень представлення (Presentation Layer). Основна мета — забезпечити взаємодію користувача з системою через веб-інтерфейс, реалізувати механізми автентифікації, реєстрації абітурієнтів, подачі заявок та перегляду рейтингів.

### Завдання:

1. Налаштувати маршрутизацію запитів через Spring MVC контролери.
2. Створити динамічні HTML-сторінки за допомогою шаблонізатора FreeMarker.
3. Забезпечити валідацію вхідних даних на рівні контролерів.
4. Реалізувати розмежування доступу до сторінок залежно від ролі користувача (ADMIN/APPLICANT).

---

## 2. Опис результатів виконання роботи

### 2.1. Веб-сторінки (Шаблони FreeMarker)

Для забезпечення єдиного стилю було використано Template Method pattern у файлі `layout.ftl`. Це дозволило винести спільні елементи (меню, підключення стилів, інформація про користувача) в один макрос.

### Основні реалізовані сторінки:

- `index.ftl` — головна сторінка з навігацією.
- `login.ftl` / `signup.ftl` — форми входу та реєстрації.
- `register.ftl` — форма реєстрації абітурієнта з введенням оцінок.
- `apply.ftl` — сторінка подачі заявки.
- `ranking.ftl` — відображення рейтингу за балами.
- `admission.ftl` — інтерфейс адміністратора для запуску алгоритму зарахування.

---

### 2.2. Контролери (Spring MVC)

Контролери виконують роль посередників між сервісним шаром та веб-сторінками.

#### AuthController
- Обробляє реєстрацію користувачів.
- Відображає форму входу.
- Використовує `AuthService` для збереження паролів із застосуванням BCrypt.

#### ApplicantController
- Приймає дані абітурієнта.
- Використовує `InputValidator` для перевірки коректності (наприклад, допустимий діапазон балів 1–12).

#### AdminController
- Доступний лише користувачам з роллю ADMIN.
- Містить метод `processAdmission`, який запускає алгоритм ранжування через `AdmissionService`.

#### GlobalExceptionHandler
- Перехоплює винятки в системі.
- Відображає дружню сторінку `error.ftl`.
- Централізує обробку помилок.

## 4. Фрагмент коду

```java
@PostMapping("/admission")
public String processAdmission(@RequestParam String facultyName, Model model) {
    try {
        AdmissionSheet sheet = admissionService.processAdmission(facultyName);

        AdmissionResultResponse result = new AdmissionResultResponse(
                facultyName,
                ResponseMapper.toResponseList(sheet.getAdmitted()),
                ResponseMapper.toResponseList(sheet.getRejected()));

        model.addAttribute("result", result);
        logger.info("Зарахування проведено для факультету: {}", facultyName);
    } catch (Exception e) {
        logger.error("Помилка зарахування", e);
        model.addAttribute("error", e.getMessage());
    }

    model.addAttribute("faculties", facultyRepository.findAll());
    return "admission";
}
```
## 5. Висновки

Під час виконання лабораторної роботи було реалізовано повноцінний рівень представлення системи.

Використання MVC забезпечило чітке розділення відповідальності.

Інтеграція Spring Security дозволила захистити адміністративний функціонал.

FreeMarker забезпечив динамічне формування сторінок.

Валідація гарантує коректність введених даних та стабільність бізнес-логіки.
