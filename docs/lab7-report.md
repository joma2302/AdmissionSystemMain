## 1. Мета роботи

Освоєння технології створення динамічних веб-сторінок на стороні сервера за допомогою шаблонізатора FreeMarker у поєднанні зі Spring MVC.

---

## 2. Теоретичні відомості та відповіді на питання

**FreeMarker** — це Java-орієнтований шаблонізатор (Template Engine), який дозволяє відокремити бізнес-логіку програми від її відображення.

* **Макроси (`<#macro>`):** використовуються для створення повторюваних фрагментів коду (наприклад, макетів сторінок).
* **Директиви управління:**

    * `<#list>` — ітерація по списках
    * `<#if>` — логічні умови
* **Об’єктна модель:** дані передаються в шаблон у вигляді Map або Model, де ключі відповідають назвам змінних у шаблоні.

---

## 3. Виконання завдання

### 3.1. Створення базового шаблону (макроси)

Для забезпечення єдиного стилю системи був створений файл `layout.ftl`, що реалізує підхід повторного використання шаблонів.

**Фрагмент коду `layout.ftl`:**

```html
<#macro page title="Приймальна комісія">
<!DOCTYPE html>
<html lang="uk">
<head>
    <title>${title}</title>
    <style>
        .navbar { background: #2c3e50; padding: 15px; display: flex; }
    </style>
</head>
<body>
    <div class="navbar">
        <a href="/">Головна</a>
        <a href="/ranking">Рейтинг</a>
    </div>

    <div class="container">
        <#nested>
    </div>
</body>
</html>
</#macro>
```

---

### 3.2. Передача даних з сервера (Model)

Для передачі даних у шаблони використовуються контролери Spring MVC.
Контролер додає атрибути до `Model`, після чого вони стають доступними у FreeMarker.

**Фрагмент `AdminController.java`:**

```java
@PostMapping("/admission")
public String processAdmission(@RequestParam String facultyName, Model model) {
    try {
        AdmissionResultResponse result = ...;
        model.addAttribute("result", result);
    } catch (Exception e) {
        model.addAttribute("error", e.getMessage());
    }
    return "admission";
}
```

---

### 3.3. Використання даних у шаблоні

На сторінці рейтингу (`ranking.ftl`) використано цикл і змінні моделі для відображення списку заявок.

**Фрагмент `ranking.ftl`:**

```html
<#import "layout.ftl" as layout>

<@layout.page title="Рейтинг заявок">
    <h1>Рейтинг на факультет: ${facultyName!""}</h1>

    <table>
        <tr>
            <th>№</th>
            <th>Абітурієнт</th>
            <th>Бали</th>
        </tr>

        <#list ranked as app>
            <tr>
                <td>${app?index + 1}</td>
                <td>${app.applicantName()}</td>
                <td>${app.totalScore()}</td>
            </tr>
        </#list>
    </table>
</@layout.page>
```
