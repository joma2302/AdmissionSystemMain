### Текст програми з коментарями (ключові фрагменти)

#### 1. Доменна модель (Алгоритм зарахування)
```java
/**
 * Клас відомості зарахування, що інкапсулює логіку ранжування.
 */
public class AdmissionSheet {
    private final Faculty faculty;
    private final List<Application> applications;

    public void determineAdmitted() {
        // Сортування за балами та перевірка вимог
        List<Application> sorted = applications.stream()
            .sorted(Comparator.comparingInt(Application::getTotalScore).reversed())
            .toList();

        int admittedCount = 0;
        for (Application app : sorted) {
            // Враховуємо ліміт місць та мінімальні бали з предметів
            if (admittedCount < faculty.getMaxStudents() && meetsRequirements(app)) {
                app.setStatus(ApplicationStatus.ADMITTED);
                admittedCount++;
            } else {
                app.setStatus(ApplicationStatus.REJECTED);
            }
        }
    }
}
```

#### 2. Інфраструктурний шар (Репозиторій MySQL)
```java
/**
 * MySQL-реалізація для роботи з абітурієнтами.
 */
@Repository
public class MySqlApplicantRepository implements ApplicantRepository {
    private final DataSource dataSource;

    @Override
    public void save(Applicant applicant) {
        String sql = "INSERT INTO applicants (id, first_name, last_name) VALUES (?, ?, ?) " +
                     "ON DUPLICATE KEY UPDATE first_name = ?, last_name = ?";
        try (Connection conn = dataSource.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql)) {
            stmt.setString(1, applicant.getId());
            stmt.setString(2, applicant.getFirstName());
            stmt.setString(3, applicant.getLastName());
            // ... виконання запиту
            stmt.executeUpdate();
        } catch (SQLException e) {
            throw new RuntimeException("DB Error", e);
        }
    }
}
```

### Вхідні та вихідні дані

**Вхідні дані:**
- Персональні дані абітурієнтів (ПІБ, ID).
- Оцінки з предметів (ЗНО/НМТ) у діапазоні 100-200 балів або 1-12.
- Параметри факультетів (назва, квота місць, предметні вимоги).
- Облікові дані для авторизації (логін/пароль).

**Вихідні дані:**
- Рейтингові списки абітурієнтів.
- Статуси заяв (Очікує, Зараховано, Відхилено).

### Змістовний аналіз та висновки

**Аналіз результатів:**
Проектування системи за принципами **Domain-Driven Design (DDD)** дозволило створити гнучку архітектуру, де бізнес-логіка відділена від технічних деталей (БД, веб). Впровадження **Spring Security** забезпечило надійний захист даних. Використання **FreeMarker** як шаблонізатора дозволило реалізувати складний динамічний інтерфейс. Автоматизація процесу зарахування через `AdmissionSheet` мінімізує людський фактор при прийнятті рішень.

**Висновки:**
1. У ході навчальної практики було розроблено повноцінну інформаційну систему «Приймальна комісія».
2. Реалізовано всі 20 етапів розробки: від проектування БД до фінального профілювання додатка.
3. Додаток підтримує рольову модель доступу та забезпечує повний цикл вступної кампанії.
4. Результати тестування (53 успішних тести) підтверджують стабільність та коректність роботи алгоритмів.
5. Система готова до розгортання та може бути масштабована для потреб реального навчального закладу.
