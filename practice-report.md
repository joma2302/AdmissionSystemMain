# Звіт про виконання завдань навчальної практики
## Тема: Розробка інформаційної системи «Приймальна комісія»



---

### 2. Вступ
**Напрям дослідження:** Проектування та розробка веб-орієнтованих інформаційних систем на базі архітектури Spring Boot з використанням реляційних баз даних.

**Мета практики:** Отримання практичних навичок у розробці повнофункціональної інформаційної системи, що автоматизує процеси роботи приймальної комісії вищого навчального закладу.

**Завдання практики:**
1. Проектування архітектури системи (шари Domain, Application, Infrastructure).
2. Реалізація механізму автентифікації та авторизації (RBAC) для ролей «Адміністратор» та «Абітурієнт».
3. Розробка функціоналу керування спеціальностями (CRUD операції).
4. Реалізація процесу реєстрації абітурієнтів та подання ними заяв на вступ.
5. Розробка алгоритму автоматичного зарахування на основі рейтингового балу.
6. Створення інтерфейсу для завантаження документів абітурієнтами.

### 3. Опис результатів виконання лабораторних робіт
Система розроблена як Spring Boot додаток. Основні етапи:
- **Lab 2-4:** Проектування предметної області (Entities: Applicant, Faculty, Application, Grade). Налаштування бази даних MySQL.
- **Lab 5-7:** Реалізація репозиторіїв (JDBC/Spring Data) та сервісного шару. Налаштування Spring Security для розмежування прав доступу.
- **Lab 8-9:** Розробка веб-інтерфейсу за допомогою шаблонізатора FreeMarker. Реалізація контролерів для основних бізнес-процесів.
- **Lab 10 (Фінальна):** Реалізація спеціалізованих функцій адміністратора (керування факультетами, перегляд списків) та абітурієнта (завантаження документів).

#### Доменна модель абітурієнта (Applicant.java)
```java
/**
 * Доменна сутність «Абітурієнт».
 * Інкапсулює дані абітурієнта та його оцінки.
 */
public class Applicant {
    private final String id;
    private final String firstName;
    private final String lastName;
    private String documentsPath; // Шлях до завантажених документів
    private final List<Grade> grades;

    // Конструктор з валідацією обов'язкових полів
    public Applicant(String id, String firstName, String lastName) {
        this.id = id;
        this.firstName = firstName;
        this.lastName = lastName;
        this.grades = new ArrayList<>();
    }

    // Метод додавання оцінки з перевіркою дублікатів
    public void addGrade(Grade grade) {
        // ... логіка перевірки ...
        grades.add(grade);
    }
    
    // Розрахунок загального балу
    public int getTotalScore() {
        return grades.stream().mapToInt(Grade::getScore).sum();
    }
}
```

### 5. Вхідні та вихідні дані
**Вхідні дані:**
- Реєстраційні дані абітурієнта (ID, ПІБ, бали ЗНО).
- Заявки абітурієнтів (вибір факультету).
- Параметри факультетів (назва, ліміт місць).
- Файли документів (PDF/JPG).

**Вихідні дані:**
- Статус заявки (PENDING, ACCEPTED, REJECTED).
- Рейтингові списки абітурієнтів.
- Відомості про зарахування по кожному факультету.

### 6. Змістовний аналіз результатів та висновки

**Аналіз проектування:**
Використання багатошарової архітектури дозволило відокремити бізнес-логіку від інфраструктури (БД, Веб). Застосування патернів Repository та Service забезпечило гнучкість системи.

**Аналіз програмування:**
Spring Boot значно прискорив розробку за рахунок автоконфігурації. Використання FreeMarker дозволило створити чистий та адаптивний UI. Безпека реалізована через Spring Security, що гарантує цілісність даних.

**Аналіз тестування:**
Проведено інтеграційне тестування репозиторіїв та сервісів. Верифіковано алгоритм зарахування: система коректно відбирає абітурієнтів з найвищими балами в межах ліміту місць.

**Висновки:**
В ході практики була розроблена інформаційна система «Приймальна комісія», яка повністю відповідає поставленим вимогам. Система автоматизує ключові аспекти вступної кампанії, мінімізуючи людський фактор при формуванні списків зарахованих та забезпечуючи зручний інтерфейс для абітурієнтів.
