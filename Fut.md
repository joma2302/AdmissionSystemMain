# Ідеї щодо розвитку проекту AdmissionSystem

Цей документ містить пропозиції щодо нових функцій та виправлень для системи «Приймальна комісія».

## Для Адміністратора (ADMIN/MANAGER)
*   [✓] **Експорт звітів:** Додати можливість завантаження списку абітурієнтів у форматі CSV/Excel для подальшого аналізу.
*   [✓] **Розширені фільтри:** Додати можливість фільтрації заявок за діапазоном балів (наприклад, "від 150 до 180"), а не лише за спеціальністю чи статусом.
*   [✓] **Панель керування користувачами:** Функціонал для розблокування облікових записів або ручного скидання паролів абітурієнтів.
*   [✓] **Візуалізація статистики:** Використання JS-бібліотек (наприклад, Chart.js) для відображення графіків наповненості факультетів на Dashboard.
*   **Логування дій:** Створення сторінки в адмін-панелі для перегляду останніх дій (audit log) користувачів (хто і коли змінив статус заявки).

## Для Абітурієнта (APPLICANT)
*   [✓] **Редагування профілю:** Дозволити зміну персональних даних (ПІБ, контактна інформація) до моменту подачі заявки.
*   [✓] **Завантаження документів:** Реалізувати функціонал завантаження скан-копій атестатів та сертифікатів (зберігання в файловій системі або S3).
*   [✓] **Історія статусів:** Відображення "timeline" змін статусів заявки (наприклад: "Подано" -> "Перевіряється" -> "Зараховано").
*   **PDF-генерація:** Додати кнопку "Завантажити заяву" у форматі PDF.
*   **Сповіщення:** Система сповіщень на пошту при зміні статусу заявки.

## Виправлення багів та технічні покращення
*   **Валідація вводу:** Додати валідацію полів на стороні клієнта та сервера (наприклад, перевірка діапазону балів 100-200).
*   **Покращення UI/UX:**
    *   Додати "темну тему" для адмін-панелі.
    *   Покращити мобільну верстку (адаптивність таблиць).
*   **Кешування:** Додати інвалідацію кешу при зміні статусу заявки, щоб дані в адмін-панелі оновлювалися миттєво.
*   **Тестування:** Покрити тестами сервісний шар (AdmissionService), оскільки зараз переважно протестовані контролери.
*   [✓] **Error Handling:** Створити більш інформативні сторінки помилок (наприклад, для помилок 404 та 403), які б містили кнопку повернення на головну.
